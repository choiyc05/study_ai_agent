## AI Agent Protocol
> 서로 다른 AI 에이전트들이 대화하기 위해 사용하는 공용 언어이자 규칙


## 1. AI 에이전트 프로토콜의 종류

현재 업계에서 표준이 되기 위해 경쟁 중인 주요 프로토콜은 다음과 같습니다.
* **Agent Protocol (by e2b & AutoGPT):** 현재 가장 널리 시도되는 표준 중 하나입니다. AI 에이전트에게 작업을 시키고, 결과를 받고, 진행 상황을 확인하는 방식을 통일하려는 프로젝트입니다.
* **MCP (Model Context Protocol - by Anthropic):** 최근 가장 뜨거운 감자입니다. AI 모델이 로컬 파일, 데이터베이스, Slack 같은 외부 도구에 안전하고 쉽게 연결될 수 있도록 돕는 표준 규격입니다.
* **OpenAI Realtime API / Assistants API:** 특정 기업의 규격이지만, 워낙 사용자가 많아 사실상의 표준(De Facto Standard) 역할을 합니다.
* **FIPA-ACL:** 사실 이건 아주 예전(멀티 에이전트 시스템 시절)부터 있던 고전적인 규약인데, 최근 다시 현대화되어 논의되기도 합니다.

---

## 2. 핵심 기능 및 주요 역할

AI 에이전트 프로토콜은 단순히 데이터를 전달하는 것을 넘어 다음과 같은 역할을 수행합니다.

### 🛠 상호운용성 (Interoperability)
서로 다른 개발자가 만든 A 에이전트(기상 분석용)와 B 에이전트(여행 계획용)가 중간 단계 없이 바로 대화하여 "비가 안 오는 날로 여행 일정을 짜줘"라는 미션을 완수하게 합니다.
### 📂 작업 관리 (Task Lifecycle)
에이전트가 수행 중인 작업이 '대기 중'인지, '실행 중'인지, 혹은 '실패'했는지를 표준화된 상태 코드로 알려줍니다. 
### 🔗 도구 연결 (Tool & Context Access)
에이전트가 내 컴퓨터의 엑셀 파일을 읽거나, 외부 웹사이트에서 정보를 긁어올 때 어떤 경로와 방식으로 접근할지 보안 규칙과 형식을 정의합니다.

---

## 3. 현 시점에서 무엇이 좋은지 추천

입문자이자 개발에 관심이 있는 학습자라면, 저는 **MCP (Model Context Protocol)**와 **Agent Protocol** 두 가지를 먼저 살펴보시는 것을 추천합니다.

| 추천 프로토콜 | 이유 | 대상 |
| :--- | :--- | :--- |
| **MCP (Anthropic)** | 현재 생태계 확장이 가장 빠르며, 로컬 데이터와 AI를 연결하는 데 강력합니다. | 실무적인 도구 연결을 배우고 싶을 때 |
| **Agent Protocol** | 구조가 매우 단순하고 REST API 기반이라 이해하기 쉽습니다. | 에이전트의 통신 기초를 다지고 싶을 때 |


---

1. Agent Protocol: "에이전트 간의 소통" (Horizontal)
2. MCP (Model Context Protocol): "에이전트와 데이터의 연결" (Vertical)

---

## IT에서의 프로토콜(Protocol)
- **통신을 하기 위한 약속(규약) / 형식이 있는 방법론**
```
구문 (Syntax): 데이터의 '형식'입니다. (예: "첫 8비트는 주소, 다음 8비트는 데이터야.")
의미 (Semantics): 그 데이터가 '무엇'을 뜻하는지입니다. (예: "100이라는 숫자가 오면 '삭제'라는 뜻이야.")
타이밍 (Timing): '언제' 보내고 얼마나 '기다릴지'입니다. (예: "내가 말하고 5초 안에 답이 없으면 끊긴 걸로 알게.")
```

### MCP Server
- 데이터를 'serve'하는 역할
```
설계도인가 - "내 PC의 특정 데이터를 어떻게 읽고 쓰는지"에 대한 규칙이 담긴 프로그램입니다.
Tool인가 - 클로드가 필요할 때 꺼내 쓰는 '특수 도구' 세트입니다.
작동 방식: 
1. 클로드(두뇌)가 "파일을 읽어야겠어"라고 판단합니다.
2. 클로드가 MCP 규격으로 명령을 보냅니다.
3. MCP 서버가 그 명령을 받아서 실제 컴퓨터 명령(예: cat memo.txt)으로 바꿔 실행합니다.
4. 그 결과물만 다시 클로드에게 전달합니다.
```
